## zk分布式锁
#### 利用zk的瞬时有序节点的特性
#### 多线程并发创建瞬时节点
#### 得到有序的序列 
#### 序号最小的线程获得锁
#### 其他线程则监听自己序号的前一个序号
####  前一个线程执行完成，删除自己序号的节点
#### 下一个序号的线程得到 通知，继续执行
####  创建节点时，已经确定了线程的执行顺序
### 瞬时节点有序 瞬时节点不可再有子节点 ，会话结束瞬时节点自动消失 
### 可设置观察器的三个方法  getData ，getChildren ，exist;
### 节点数据发生变化 发送给客户端
### 观察器 只能监控一次，再监控需要重新设置
